# 聊天室系统需求分析文档

## 1. 项目概述

### 1.1 项目背景
构建一个基于 Node.js 的实时 Web 聊天室应用，支持多房间管理、用户实时通信、管理员后台管理等功能。

### 1.2 项目目标
- 提供简洁易用的实时聊天功能
- 支持多聊天室隔离管理
- 实现完善的安全防护机制
- 提供管理员后台管理界面
- 无需数据库，基于文件系统存储

### 1.3 技术栈
- **前端**: Bootstrap 5.3.0, 原生 JavaScript
- **后端**: Node.js (原生), WebSocket
- **存储**: JSONL 文件格式
- **日志**: Winston (日志轮转)
- **安全**: bcrypt (密码加密), he (XSS 防护)

## 2. 功能需求

### 2.1 用户端功能

#### 2.1.1 房间访问
- **邀请码验证**: 用户必须输入 6 位邀请码才能进入聊天室
- **房间隔离**: 不同房间的用户和消息完全隔离
- **刷新重验证**: 每次刷新页面都需要重新输入邀请码

#### 2.1.2 用户身份管理
- **IP 识别**: 基于 IP 地址识别用户身份
- **昵称生成**:
  - 首次进入可手动输入昵称
  - 不输入则自动生成中文昵称（形容词 + 动物名）
  - 历史用户自动加载历史昵称
  - 昵称重复自动添加数字后缀
- **头像生成**: 使用 DiceBear API 自动生成矢量头像
- **会话保持**: 同一 IP 的历史记录自动恢复

#### 2.1.3 连接管理
- **单连接限制**: 同一 IP 在同一房间只能保持一个连接
- **重复连接处理**: 新连接自动踢出旧连接
- **自动重连**: 断线后自动重连（最多 5 次，指数退避）
- **心跳机制**: 每 30 秒发送心跳包保持连接

#### 2.1.4 消息功能
- **文本消息**:
  - 最大长度 1000 字符
  - 支持中英文及特殊字符
  - HTML 实体编码防止 XSS
- **图片消息**:
  - 支持粘贴图片上传（最大 5MB）
  - 支持发送图片 URL（最大 500 字符）
  - 图片自动渲染显示
  - 按日期存储在 `/data/images/YYYY-MM-DD/` 目录

#### 2.1.5 消息历史
- **初始加载**: 进入房间加载最近 50 条消息
- **历史加载**: 点击"加载更多消息"按钮加载更早消息
- **每日刷新**: 用户只能看到当天的聊天记录
- **轮询更新**: 每 5 秒轮询一次新消息

#### 2.1.6 在线状态
- **在线人数**: 实时显示当前房间在线人数
- **人数限制**: 每个房间可配置最大在线人数（默认 20）
- **满员拒绝**: 达到人数上限后拒绝新用户加入

### 2.2 管理员功能

#### 2.2.1 身份认证
- **登录验证**: 用户名密码登录（密码 bcrypt 加密）
- **会话管理**: 登录后获得 24 小时有效期的 session
- **登录保护**:
  - 最多 5 次登录尝试
  - 失败后锁定 15 分钟
- **权限分级**:
  - `admin` 账户拥有完整权限
  - 普通管理员只能修改自己的密码

#### 2.2.2 房间管理
- **创建房间**:
  - 输入房间名称
  - 设置最大用户数
  - 自动生成 6 位邀请码（字母数字，排除易混淆字符）
- **查看房间**:
  - 列出所有房间
  - 显示实时在线人数
  - 显示房间状态（启用/禁用）
- **启用/禁用房间**:
  - 禁用时踢出所有在线用户
  - 禁用后用户无法进入
- **删除房间**:
  - 永久删除房间及其所有数据
  - 踢出所有在线用户

#### 2.2.3 账户管理
- **创建管理员**:
  - 仅 `admin` 账户可创建
  - 用户名规则：字母数字下划线连字符，最大 50 字符
  - 密码规则：6-100 字符
- **修改密码**:
  - `admin` 可修改任何账户密码
  - 普通管理员只能修改自己的密码
- **删除管理员**:
  - 仅 `admin` 账户可删除
  - `admin` 账户不可删除

#### 2.2.4 日志审计
- **系统日志**: 查看应用运行日志
- **聊天记录**: 查看所有房间的聊天记录
- **日志过滤**: 支持按日期、关键词过滤

### 2.3 安全需求

#### 2.3.1 XSS 防护
- 所有用户输入使用 `he` 库进行 HTML 实体编码
- 图片 URL 验证
- CSP 安全头设置

#### 2.3.2 速率限制
- **消息限制**: 单 IP 每 3 分钟最多 20 条消息
- **轮询限制**: 单 IP 每 5 秒最多 3 次轮询请求
- **上传限制**: 单 IP 每天最多上传 100MB

#### 2.3.3 自动封禁
- **消息封禁**: 超出消息速率限制封禁 30 分钟
- **轮询封禁**: 超出轮询速率限制封禁 10 分钟
- **IP 黑名单**: 自动记录和解除封禁

#### 2.3.4 管理员安全
- 密码 bcrypt 加密存储
- 登录失败次数限制
- 会话超时机制
- 防止暴力破解

#### 2.3.5 上传安全
- 文件类型验证（仅允许图片）
- 文件大小限制（5MB）
- 每日上传配额（100MB/IP）
- UUID 文件名防止路径遍历

### 2.4 数据存储需求

#### 2.4.1 消息存储
- 格式: JSONL（每行一个 JSON 对象）
- 路径: `/data/messages/YYYY-MM-DD.jsonl`
- 按日期分文件存储
- 包含字段: 时间戳、房间 ID、用户信息、消息类型、内容

#### 2.4.2 用户存储
- 格式: JSONL
- 路径: `/data/users.jsonl`
- 记录: IP、昵称、首次访问时间

#### 2.4.3 房间存储
- 格式: JSONL
- 路径: `/data/rooms.jsonl`
- 包含: 房间 ID、名称、邀请码、最大用户数、状态、创建时间

#### 2.4.4 管理员存储
- 格式: JSONL
- 路径: `/data/admins.jsonl`
- 包含: 用户名、bcrypt 加密密码、创建时间

#### 2.4.5 图片存储
- 路径: `/data/images/YYYY-MM-DD/`
- 文件名: UUID + 扩展名
- 按日期分目录存储

#### 2.4.6 日志存储
- 格式: Winston 日志格式
- 路径: `/logs/chatroom-YYYY-MM-DD.log`
- 轮转策略:
  - 单文件最大 10MB
  - 保留最近 5 天
  - 自动删除过期日志

### 2.5 性能需求

#### 2.5.1 并发支持
- 每个房间支持最多 20 个并发用户（可配置）
- 支持多个房间同时运行
- WebSocket 连接池管理

#### 2.5.2 响应时间
- 消息广播延迟 < 100ms
- 页面加载时间 < 2s
- 历史消息加载 < 500ms

#### 2.5.3 资源限制
- 内存消息缓存: 每房间最多 50 条
- 日志文件: 最大 50MB（5 个 10MB 文件）
- 上传配额: 100MB/IP/天

## 3. 非功能需求

### 3.1 可用性
- 7x24 小时运行
- 自动重连机制
- 优雅的错误提示

### 3.2 可维护性
- 模块化架构设计
- 清晰的代码注释
- 完整的日志记录

### 3.3 可扩展性
- 支持动态创建房间
- 可配置的限制参数
- 插件化的存储接口

### 3.4 兼容性
- 支持现代浏览器（Chrome, Firefox, Safari, Edge）

## 4. 约束条件

### 4.1 技术约束
- 不使用数据库
- 使用原生 Node.js（无 Express 等框架）
- 基于文件系统存储

### 4.2 业务约束
- 用户无需注册
- 基于 IP 识别用户
- 消息不加密存储（仅传输加密）

### 4.3 安全约束
- 必须通过邀请码访问
- 严格的速率限制
- 管理员密码强度要求

## 5. 用户场景

### 5.1 普通用户场景

#### 场景 1: 首次进入聊天室
1. 用户访问网站首页
2. 输入 6 位邀请码
3. 系统验证邀请码有效性
4. 提示输入昵称（可选）
5. 进入聊天室，加载历史消息
6. 开始聊天

#### 场景 2: 老用户重新进入
1. 用户访问网站首页
2. 输入邀请码
3. 系统识别 IP，自动恢复历史昵称和头像
4. 直接进入聊天室

#### 场景 3: 发送消息
1. 用户在输入框输入文本
2. 按回车或点击发送
3. 消息实时广播给房间内所有用户
4. 消息持久化到文件

#### 场景 4: 发送图片
1. 用户粘贴图片或输入图片 URL
2. 图片上传到服务器（如果是本地图片）
3. 图片消息广播给所有用户
4. 聊天窗口自动渲染图片

#### 场景 5: 查看历史消息
1. 用户滚动到聊天记录顶部
2. 点击"加载更多消息"按钮
3. 系统加载更早的消息
4. 消息插入到聊天记录顶部

### 5.2 管理员场景

#### 场景 1: 管理员登录
1. 访问 `/admin` 页面
2. 输入用户名和密码
3. 系统验证身份
4. 登录成功，进入管理后台

#### 场景 2: 创建聊天室
1. 点击"创建房间"按钮
2. 输入房间名称和最大用户数
3. 系统生成邀请码
4. 房间创建成功，显示在列表中

#### 场景 3: 管理房间
1. 查看所有房间列表
2. 查看每个房间的在线人数
3. 启用/禁用房间
4. 删除不需要的房间

#### 场景 4: 查看日志
1. 切换到日志审计标签
2. 选择日期范围
3. 查看系统日志或聊天记录
4. 搜索特定关键词

#### 场景 5: 管理账户
1. 切换到账户管理标签
2. 创建新管理员账户（仅 admin）
3. 修改密码
4. 删除管理员账户（仅 admin）

## 6. 风险分析

### 6.1 技术风险
- **WebSocket 连接稳定性**: 通过心跳和自动重连机制缓解
- **文件系统性能**: 通过 JSONL 追加写入和日志轮转优化
- **并发写入冲突**: 使用文件锁或队列机制

### 6.2 安全风险
- **DDoS 攻击**: 通过速率限制和 IP 封禁防护
- **XSS 攻击**: 通过 HTML 实体编码防护
- **暴力破解**: 通过登录次数限制和锁定机制防护

### 6.3 业务风险
- **存储空间不足**: 通过日志轮转和定期清理控制
- **恶意用户**: 通过速率限制和封禁机制管理
- **房间滥用**: 通过管理员审核和删除功能控制
